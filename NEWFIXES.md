# New Fixes

This document is the final four-agent ownership brief for the full hardening and production refactor pass. The whole surface is covered between Agents 1 through 4 with no shared “misc” bucket, no fifth cleanup lane, and no fallback compatibility work. Every concern belongs to exactly one owner, and any future follow-up should preserve these boundaries so we do not reintroduce giant files, duplicated trust logic, or CLI/runtime drift.

## Agent 1

You own artifact trust, selector semantics, wrapper validation, sidecar identity, and every read path that turns files on disk into product truth. Your job is to ensure that `artifacts`, `memory`, `profile`, `report`, and `ci` all consume one coherent validated-bundle model, with explicit separation between direct-trace inspection and validated wrapper-backed inspection. You are responsible for checksum/header enforcement, manifest/report/trace identity coherence, selector alias parity, stale-sidecar rejection or refresh policy, cached artifact parsing where it improves cost without weakening strictness, and the removal of accidental file-to-parent fallback behavior. Do not let command-specific trust rules creep back in, do not let profile/report reads mutate state implicitly, and do not allow any path where a parseable wrapper is treated as trusted without the same structural validation the rest of the product enforces.

## Agent 2

You own runtime execution, finalization order, persisted run truth, and the deterministic execution contract across `run`, `test`, `fuzz`, `explore`, `replay`, `shrink`, and `doctor`. Your job is to keep one canonical final summary and one canonical identity before report, manifest, reporter, profile, memory, and trace artifacts are written, and to make replay consume the same execution truth rather than a drifted reimplementation. You are responsible for deterministic suite seeds and ordering, aggregate-to-child trace lineage, exact shrink failure-class preservation, runtime-owned profile artifact generation, final findings/timing coherence across all outputs, and keeping host-backed and deterministic execution paths behaviorally aligned. If a command layer starts owning producer logic again, pull it back into runtime-owned finalization rather than letting persistence semantics split.

## Agent 3

You own the public contract: Clap shape, parse-time validity, workflow commands, machine-readable output, public crate exports, help/schema text, and reporting semantics. Your job is to make the CLI tell the truth and reject impossible states before dispatch, not after; that includes narrowing crate-root exports, aligning help text with real selector behavior, keeping schema/usage/help in sync with the actual DSL and command model, making CI always return structured output on failure, and ensuring workflow surfaces like `gate` and `full` prove the real product contract instead of a narrower library-only path. You are also responsible for truthful reporting terminology and behavior, including keeping query/path semantics explicit, making cheap report surfaces stay cheap by default, surfacing profile diagnosis failures rather than swallowing them, and avoiding any flag or mode name that promises behavior the implementation does not actually deliver.

## Agent 4

You own end-to-end proof, SDK parity, repository hygiene, and structural production layout. Your job is to keep regression coverage locked around selector parity, stale-sidecar recovery, timing-field coherence, host-backed behavior, recorded trace integrity, and SDK agreement with the Rust CLI, while also making sure the repository stays physically maintainable. That includes splitting oversized production modules into concern-owned files, moving embedded mega-tests out of hot production paths, breaking parity suites into focused files, keeping file names clean and single-purpose where collisions do not require more specificity, and preserving semantics while decomposing large modules like workflow orchestration, runtime internals, and command surfaces. You also own removal of stray tracked workspace noise and any test or support-file cleanup needed so future engineers can land changes in the right place without reopening monoliths.
